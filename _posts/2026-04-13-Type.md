## TypeScript의 타입
TypeScript의 기본 타입은 JavaScript의 원시타입(primitive type)과 동일하게 7가지
```
null
undefined
boolean (true, false)
string
number
bigint
symbol
```


## 원시타입과 객체의 차이
`string`과 `String`은 다름.  
`string`은 순수하게 타입(자료형)을 알려주는 **원시타입(Primitive)** 이고 `String`은 **래퍼객체(Wrapper Object)** 로 메소드, 속성 등을 포함.  
TypeScript에서는 `string`과 같이 타입의 형태로 쓰는 것을 권장.<br>
어차피 `string`으로 선언한 변수에서 메소드를 호출해도, JavaScript 엔진이 알아서 `String`으로 변환하여 실행하고, 다시 `string`으로 돌려놓음.


## 그럼 `String`은 왜 필요함?
어차피 원시 타입으로 써도 다 되게 할거면 String이 존재할 필요가 있나?  그냥 python이나 c처럼 함수를 따로 빼면 되는거 아닌가?  
이는 TypeScript의 원형인 JavaScript 때문.  
Java는 객체지향을 추구하여 원시타입을 객체처럼 쓰고 싶었고, Wrapper Object가 탄생함.  
JavaScript는 이름에서 알 수 있듯이 java의 개념과 문법을 유사하게 만듬.  
TypeScript는 JavaScript의 superset이라 걍 남게 됨.  


**필요해서 만든게 아니라 원래 있던게 남아있는 것.**
