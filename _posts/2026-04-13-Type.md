## TypeScript의 타입
TypeScript의 기본 타입은 JavaScript의 원시타입(primary type)과 동일하게 7가지
```
null
undefined
boolean (true, false)
string
number
bigint
symbol
```
<br>

## 원시타입과 객체의 차이
`string`과 `String`은 다름. <br>
`string`은 순수하게 타입(자료형)을 알려주는 원시타입이고 `String`은 객체(Object)로 메소드, 속성 등을 포함.<br>
TypeScript에서는 `string`과 같이 타입의 형태로 쓰는 것을 권장. 어차피 `string`으로 선언한 변수에서 메소드를 호출해도, TypeScript가 알아서 `String`으로 변환하여 실행하고, 다시 `string`으로 돌려놓음.<br><br>

## 그럼 `String`은 왜 필요함?
어차피 string으로 써도 다 되게 할거면 String이라는 무거운 객체를 굳이 만들 필요가 있었나?  그냥 python이나 c처럼 함수를 따로 빼면 되는거 아닌가? <br>
이는 TypeScript의 원형인 JavaScript 때문. <br>
JavaScript는 OOP에 대한 강박 때문인지 자료형과 메소드를 분리하지 않고 하나에 다 때려박았음. <br>
JavaScript는 Java의 문법, 이름 등과 유사하게 개발되었는데, Java가 원시 자료형과 객체를 나눠 쓰는 복잡한 시스템을 가지고 있었음.<br>
TypeScript는 JavaScript에 몇 가지 개념을 추가한 것이라 걍 남게 됨. <br>
